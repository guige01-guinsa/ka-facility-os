# 운영 콘솔 1페이지 시작 가이드

대상 화면: `/web/console`  
운영 가이드 URL: `/web/console/guide`

## 목적

신규 사용자가 운영 콘솔에서 아래 4가지를 바로 할 수 있도록 만든 빠른 시작 가이드입니다.

1. 관리자 토큰 연결 확인
2. 점검 목록 조회
3. 작업지시 목록 조회
4. 월간 리포트 조회/다운로드

## 5분 시작 순서

1. `1) 인증 연결`에서 `X-Admin-Token` 입력
2. `토큰 저장` 클릭
3. `연결 테스트 (/api/auth/me)` 클릭
4. `3) 운영 데이터 HTML 조회`에서 아래 순서대로 실행
5. `공개: 서비스 정보`
6. `인증: 내 권한 확인`
7. `점검 목록`
8. `작업지시 목록`
9. `운영 대시보드 요약`
10. `핸드오버 브리프`
11. `월간 감사 리포트`

## 입력값 읽는 법

- `site`: 선택값. 비워두면 전체 조회
- `limit`: 기본 `20`. 신규 사용자는 그대로 사용
- `offset`: 기본 `0`. 다음 페이지 볼 때만 사용
- `status`: 작업지시 상태. 예: `open`, `acked`
- `days`: 대시보드 집계 기간. 기본 `30`
- `month`: 월간 리포트 기준월. `YYYY-MM`

## 일일 권장 순서

1. `연결 테스트 (/api/auth/me)`로 토큰 상태 확인
2. `점검 목록`으로 오늘 점검 등록 여부 확인
3. `작업지시 목록`으로 미처리 건 확인
4. `운영 대시보드 요약`으로 누락/지연 확인
5. `핸드오버 브리프`로 인수인계 이슈 정리
6. 월말/월초에는 `월간 감사 리포트` 조회 후 `HTML/CSV/PDF` 다운로드

## 자주 보는 오류

- `토큰이 없어 조회를 실행할 수 없습니다.`
  - `토큰 저장`을 먼저 실행
- `HTTP 401` 또는 `403`
  - 토큰 만료, 권한 부족, site scope 불일치 가능성
  - 먼저 `/api/auth/me` 결과 확인
- `결과 배열이 비어 있습니다.`
  - 현재 조건에 맞는 데이터가 없음
  - `site`, `status`, `month`, `offset` 재확인
- 월간 리포트가 비어 보일 때
  - `month`가 `YYYY-MM` 형식인지 확인

## 운영 원칙

- 처음에는 `site`를 비우고 전체 조회로 구조를 익힌다.
- 결과가 많아질 때만 `site`를 넣어 좁힌다.
- 보고서 다운로드 전에는 먼저 `JSON 조회`로 결과를 확인한다.
- 권한 오류가 나면 화면을 의심하기보다 토큰/역할/site scope를 먼저 본다.
